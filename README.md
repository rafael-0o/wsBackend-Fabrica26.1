# Rick & Morty Squad Manager

Django + DRF API to build a squad with Rick and Morty characters.

The project separates:
- External data (name, image, species, status): [Rick and Morty API](https://rickandmortyapi.com/)
- Private database data (per user): `role` and `tactical_note`

---

## Features

- User registration
- JWT Authentication
- Create, list, edit, and remove squad members
- Consult enriched squad with full character data
- Simple web pages to test the flow without Postman

---

## Stack

- Python + Django `6.0.3`
- Django REST Framework `3.16.1`
- JWT with `djangorestframework-simplejwt`
- `requests` to consume the external API
- Database:
  - SQLite (simple local dev)
  - PostgreSQL (via Docker)

---

## Main Model

`SquadMember` stores:
- `user`
- `character_id`
- `role` (`LEAD | TANK | SUPP | RECON`)
- `tactical_note`
- `created_at`

Important Rule:
- The combination of `user + character_id` is unique (`UniqueConstraint`)
- A user cannot recruit the same character twice

- `squadManager/viewSet.py`: REST endpoints (API)

## Endpoints

Local Base URL: `http://localhost:8000`

For protected endpoints, send:

`Authorization: Bearer <access_token>`

### Auth

- `POST /api/auth/register/` -> Create user
- `POST /api/auth/login/` -> Returns `access` + `refresh` tokens

Example (`/api/auth/login/`):

```json
{
  "username": "your_user",
  "password": "your_password"
}
```

### Squad (JWT)

- `GET /api/squad/`
- `POST /api/squad/`
- `GET /api/squad/<id>/`
- `PATCH /api/squad/<id>/`
- `DELETE /api/squad/<id>/`
- `GET /api/squad/enriched/`

Creation Example:

```json
{
  "character_id": 1,
  "role": "RECON",
  "tactical_note": "High mobility."
}
```

### Web Pages

- `GET /api/web/` -> Home
- `GET /api/web/register/` -> Register
- `GET /api/web/squad/` -> User's squad

## How to Run (Docker + Postgres)

Prerequisite: Docker Desktop running.

```bash
docker compose up --build
```

The API runs at `http://localhost:8000` and executes `migrate` automatically.

---

## How to Run (Local, SQLite)

1. Create virtual environment

```bash
python -m venv venv
```

2. Activate virtual environment

- Windows (PowerShell):

```bash
venv\Scripts\Activate.ps1
```

3. Install dependencies

```bash
pip install -r requirements.txt
```

4. Run migrations

```bash
python manage.py migrate
```

5. (Optional) Create superuser

```bash
python manage.py createsuperuser
```

6. Start server

```bash
python manage.py runserver
```

---

## Quick Test Flow

1. Create user at `/api/auth/register/`
2. Obtain token at `/api/auth/login/`
3. Use token for `POST /api/squad/`
4. Open `/api/squad/enriched/` to see full character data
5. Test UI at `/api/web/`

## Environment Variables

Main:
- `DJANGO_SECRET_KEY`
- `DJANGO_DEBUG`
- `DJANGO_ALLOWED_HOSTS`
- `POSTGRES_DB`
- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `POSTGRES_HOST`
- `POSTGRES_PORT`

Note: When `POSTGRES_DB` exists in the environment, the project uses PostgreSQL; otherwise, it uses SQLite.
